Known issues in travelrecce

last revised 31st August, 2021

travelrecce is a very new service, in the next few months we're going to be learning about all the cool ways our users are able to break it.

These issues will be documented here, with the aim of showing our users what we're actively working on fixing.


    1st September, 2021
    Found in v1.0.2 (current version)

  • When updating a trips details, said details are not updated across the application. Requires a refresh or restart to load changes.

    Found by Squared


    31st August, 2021
    Found in v1.0.1 (first version)

  • Sometimes when opening another user's trip from said user's profile page, the (current) user is served a grey (empty) screen.

    Found by Squared

    Fixed in v1.0.2


  • Depending on the number of trips in the user's feed, a trip's corresponding username may be false. This number is usually 2, but may happen with different quantities.

    Found by Squared

    Fixed in v1.0.2


  • The entries of a trips 'liked by' page may take the user to an incorrect profile page.

    Found by Squared

    Fixed in v1.0.2


  • Maps will sometimes load in disregard for the user's device brightness (dark mode, light mode). This makes an application in dark mode load a map profile optimised for light mode, aesthetically unpleasing.

    Found by Squared

  • When swiping between the user's profile and the application settings page, the user's profile blur matrix is not properly masked. This blur restricts areas of the settings page while moving between pages, disappears after reaching the page.

    Found by Squared
